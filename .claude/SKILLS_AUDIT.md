# Skills Audit

Date: 2026-07-14
Scope: Kotlin Multiplatform + Compose Multiplatform skill integration for this repo.

All candidate repos were shallow-cloned into a scratch directory (outside the project) and inspected file-by-file before anything was copied into `.claude/skills/`. Nothing was installed sight-unseen. `.claude/skills/` did not previously exist, so no backup was needed.

Audit checklist applied to every candidate: frontmatter (`allowed-tools`, hooks, dynamic commands), embedded scripts (network calls, `exec`/`eval`, `subprocess`), install scripts (`curl | bash` patterns), secret/`.env`/credential access, prompt-injection-style phrasing ("ignore previous instructions", etc.), and redundancy against skills already selected.

## Custom skills authored for this repo

- **`project-context-map`** — content matches the spec given for this task exactly (four `docs/AI_*.md` outputs, token-conscious scan order, avoid-list for build/binary/generated dirs).
- **`token-context-savings`** — new, since no candidate repo offered a dedicated context-economy skill. Covers targeted reads over full-file dumps, scoped builds over full `./gradlew build`, delegating broad exploration to subagents, and summarizing rather than pasting raw output. Deliberately scoped as a *tactical* complement to `project-context-map`'s *durable documentation* role, so the two don't overlap.
